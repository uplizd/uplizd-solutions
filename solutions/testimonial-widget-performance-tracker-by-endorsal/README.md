## Quick Start
### 1) Import the Flow into Uplizd
1. Click the "Run on Uplizd" button to open the workflow in the builder.
2. Connect your required API credentials for your testimonial widget provider and analytics platform.
3. Configure the trigger interval (e.g., daily or weekly) for performance reporting.
4. Ensure nodes are correctly mapped from Chat Input to the Agent and the final Output.

### 2) Setup the Nodes
* **Chat Input**: Receives the request to initiate a performance audit or retrieve specific widget metrics.
* **Agent**: Processes the data, interprets engagement trends, and formulates optimization recommendations.
* **Composio Toolset**: Executes API calls to fetch widget data and push configuration updates.
* **Chat Output**: Delivers the final performance summary and actionable insights to your team.

### 3) Run the Flow
Use the Playground to test the workflow with prompts such as:
* `Analyze the performance of the 'Homepage-Main' testimonial widget over the last 7 days.`
* `Which testimonials have the highest click-through rate this month?`
* `Identify any underperforming widgets that need a content refresh.`

## Configuration
### 1) Language Model (Agent Node)
The agent acts as a data analyst and optimization strategist.
* Instruct the agent to prioritize high-conversion metrics.
* Configure the tone to be professional, data-driven, and actionable.
* Set the agent to flag significant performance anomalies for manual review.

### 2) Composio Toolset Node
* Provide your API key for the testimonial widget provider.
* Ensure the connection scope includes read access for analytics and write access for widget configuration.

### 3) Tool Availability
* **Analytics Fetcher**: Retrieves raw engagement data from your widget platform.
* **Widget Manager**: Updates testimonial ordering and visibility settings.
* **Notification Service**: Sends alerts to Slack or email when performance thresholds are met.
