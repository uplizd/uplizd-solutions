## Quick Start
### 1) Import the Flow into Uplizd
1. Navigate to the Uplizd marketplace and locate the Emissions Compliance Monitor.
2. Click "Import" to add the workflow template to your workspace.
3. Connect your Corrently API credentials within the integration settings.
4. Ensure nodes are correctly linked from Chat Input to the Agent and finally to the Output node.

### 2) Setup the Nodes
- **Chat Input**: Receives natural language queries regarding emissions status or report requests.
- **Agent**: Processes environmental data and applies compliance logic to determine status.
- **Composio Toolset**: Executes secure API calls to fetch and verify emissions data from external sources.
- **Chat Output**: Delivers clear, actionable summaries or generated compliance reports to the user.

### 3) Run the Flow
Use the Playground to test the agent with the following prompts:
- `Generate a summary report of our emissions compliance status for the last 30 days.`
- `Are there any current alerts regarding our carbon emission thresholds?`
- `List all recent data points retrieved from the Corrently integration.`

## Configuration
### 1) Language Model (Agent Node)
The agent acts as a compliance specialist, interpreting raw data into human-readable insights.
- Prioritize accuracy and adherence to regulatory terminology.
- Maintain a neutral, professional tone for all generated reports.
- Flag any data discrepancies or missing sensor inputs immediately.

### 2) Composio Toolset Node
Requires a valid API key for the Corrently integration and appropriate read-access scopes to fetch environmental metrics.

### 3) Tool Availability
- **Data Fetcher**: Retrieves real-time and historical emissions data.
- **Alerting Engine**: Monitors thresholds and triggers notifications.
- **Report Generator**: Formats data into standard compliance documentation.
