## Quick Start
### 1) Import the Flow into Uplizd
1. Click the "Run on Uplizd" button above to access the template.
2. Import the workflow JSON into your Uplizd workspace.
3. Connect your preferred CRM or data source via the Composio Toolset.
4. Ensure nodes are correctly mapped to your specific API endpoints and notification widgets.

### 2) Setup the Nodes
- **Chat Input**: Receives manual triggers or automated event signals from your marketing stack.
- **Agent**: Processes incoming data and determines the relevance of the social proof event.
- **Composio Toolset**: Executes the API calls to update your notification widget or marketing platform.
- **Chat Output**: Confirms successful synchronization and logs the triggered notification event.

### 3) Run the Flow
Use the Playground to test your notification triggers:
- `Trigger a recent purchase notification for the last 5 minutes of activity.`
- `Show social proof for new sign-ups in the North American region.`
- `List all active social proof events currently being pushed to the website.`

## Configuration
### 1) Language Model (Agent Node)
The agent acts as the decision engine for your social proof strategy.
- Focus on identifying high-value user actions from raw event logs.
- Prioritize recency and relevance when selecting data for display.
- Maintain a professional and encouraging tone for all notification copy.

### 2) Composio Toolset Node
- Provide your API key for the target notification platform (e.g., Fomo, Proof, or custom webhook).
- Ensure the connection scope includes read access to your CRM and write access to your notification service.

### 3) Tool Availability
- **CRM Connector**: For fetching user activity and purchase history.
- **Notification API**: For pushing formatted social proof data to your front-end.
- **Analytics Bridge**: For logging notification performance and click-through data.
