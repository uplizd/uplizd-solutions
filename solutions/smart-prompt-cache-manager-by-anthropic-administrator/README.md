## Quick Start
### 1) Import the Flow into Uplizd
1. Click the "Run on Uplizd" button to open the workflow template in your dashboard.
2. Connect your preferred LLM provider and database credentials within the Uplizd environment.
3. Configure the environment variables to set your cache expiration and hit-rate thresholds.
4. Ensure nodes are correctly linked from **Chat Input** through the **Agent** to the **Composio Toolset** and finally the **Chat Output**.

### 2) Setup the Nodes
* **Chat Input:** Receives the user prompt and metadata.
* **Agent:** Analyzes the prompt to determine if a cached version exists or if a new generation is required.
* **Composio Toolset:** Executes the read/write operations to your cache storage.
* **Chat Output:** Returns the cached response or the newly generated content to the user.

### 3) Run the Flow
Use the Playground to test the caching logic with these prompts:
* `What is our company return policy?`
* `Summarize the technical documentation for the API v2.0.`
* `Generate a boilerplate response for a new user onboarding email.`

## Configuration
### 1) Language Model (Agent Node)
The agent acts as the decision engine for cache management.
* Use a model with strong instruction-following capabilities (e.g., Claude 3.5 Sonnet).
* Set system instructions to prioritize cache retrieval before triggering an external API call.
* Ensure the agent is configured to output structured data for cache key generation.

### 2) Composio Toolset Node
* Provide your API key to enable secure access to your storage infrastructure.
* Set the connection scope to allow the agent to read from and write to your specific cache database.

### 3) Tool Availability
* **Cache Query Tool:** Checks for existing responses based on prompt hash.
* **Cache Update Tool:** Saves new prompt-response pairs to the database.
* **Analytics Logger:** Tracks performance metrics for reporting and optimization.
