## Quick Start
### 1) Import the Flow into Uplizd
1. Navigate to the Uplizd dashboard and select "Create New Flow."
2. Import the Airtable Data Explorer template from the solution library.
3. Connect your Airtable account via the Composio integration settings.
4. Ensure nodes are correctly linked: **Chat Input → Agent → Composio Toolset → Chat Output**.

### 2) Setup the Nodes
*   **Chat Input**: Receives your natural language request regarding the database.
*   **Agent**: Processes the intent and determines the necessary Airtable operations.
*   **Composio Toolset**: Executes secure API calls to fetch, filter, or update your Airtable data.
*   **Chat Output**: Delivers the structured response or report back to the user.

### 3) Run the Flow
Use the Playground to test your database queries:
*   `"Summarize all high-priority tasks in the Marketing base for this week."`
*   `"Find all records in the 'Leads' table that haven't been updated in 30 days."`
*   `"Create a report of total revenue generated by project category from the Sales base."`

## Configuration
### 1) Language Model (Agent Node)
The agent acts as the bridge between user intent and database actions.
*   Use a model with strong reasoning capabilities (e.g., GPT-4o or Claude 3.5 Sonnet).
*   Provide clear instructions on the structure of your Airtable bases.
*   Ensure the agent is instructed to prioritize data accuracy and handle empty result sets gracefully.

### 2) Composio Toolset Node
*   Requires a valid Airtable API key or OAuth connection established within Composio.
*   Scope the connection to specific bases to ensure the principle of least privilege.

### 3) Tool Availability
*   `airtable_list_records`: Retrieve data from specific views or tables.
*   `airtable_get_record`: Fetch details for a specific record ID.
*   `airtable_update_record`: Modify existing data based on agent findings.
*   `airtable_search_records`: Perform filtered queries across the database.
