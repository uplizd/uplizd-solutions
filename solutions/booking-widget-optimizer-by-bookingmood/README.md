## Quick Start
### 1) Import the Flow into Uplizd
1. Click the "Run on Uplizd" badge above to navigate to the solution template.
2. Select "Import" to add the Booking Widget Optimizer workflow to your Uplizd workspace.
3. Authenticate your booking platform and CRM connections via the Composio integration menu.
4. Ensure nodes are correctly mapped to your specific widget IDs and API endpoints before activating the flow.

### 2) Setup the Nodes
- **Chat Input**: Receives performance triggers or manual optimization requests.
- **Agent**: Analyzes widget data and determines the optimal configuration adjustments.
- **Composio Toolset**: Executes the necessary API calls to update your booking widget settings.
- **Chat Output**: Confirms the changes applied and provides a summary of the expected impact.

### 3) Run the Flow
Use the Playground to test your configuration:
- `Analyze the performance of the primary booking widget on the pricing page and suggest optimizations.`
- `Update the booking widget configuration to prioritize morning slots for the next 48 hours.`
- `Run a diagnostic check on the booking widget and report any drop-off patterns identified in the last 7 days.`

## Configuration
### 1) Language Model (Agent Node)
The Agent acts as your optimization engine, interpreting performance metrics and applying logic.
- Use a high-reasoning model (e.g., GPT-4o or Claude 3.5 Sonnet) for accurate data interpretation.
- Instruct the agent to prioritize conversion-focused changes over aesthetic adjustments.
- Maintain a strict output format for tool calls to ensure seamless integration with your booking platform.

### 2) Composio Toolset Node
- Provide your API keys for the specific booking platform (e.g., Bookingmood, Calendly, or similar).
- Ensure the connection scope includes `read` and `write` permissions for widget settings and analytics data.

### 3) Tool Availability
- **Analytics Connector**: Fetches interaction logs and conversion rates.
- **Widget Manager**: Executes updates to widget display, availability, and form fields.
- **Notification Service**: Alerts the team when significant configuration changes are deployed.
