## Quick Start
### 1) Import the Flow into Uplizd
1. Click the "Run on Uplizd" badge above to open the template in your dashboard.
2. Connect your PhantomBuster API key to the Composio Toolset node.
3. Configure your notification channels (e.g., Slack or Email) in the Chat Output node.
4. Ensure nodes are correctly mapped to your specific PhantomBuster agent IDs and execution preferences.

### 2) Setup the Nodes
- **Chat Input**: Receives manual triggers or scheduled interval signals to initiate the health check.
- **Agent**: Analyzes the current state of your agents against historical performance benchmarks.
- **Composio Toolset**: Executes API calls to fetch logs, status updates, and trigger agent restarts.
- **Chat Output**: Delivers status reports and error resolution summaries to your preferred communication platform.

### 3) Run the Flow
Use these prompts in the Playground to test your configuration:
- `Check the status of all active PhantomBuster agents and report any failures.`
- `Restart the 'LinkedIn Lead Scraper' agent and provide a summary of the latest execution logs.`
- `Identify agents that have failed more than three times in the last 24 hours.`

## Configuration
### 1) Language Model (Agent Node)
The Agent node acts as the decision-making engine for your monitoring workflow.
- Use a high-reasoning model (e.g., GPT-4o) to accurately interpret log files.
- Instruct the agent to prioritize "Critical" failures over "Warning" status updates.
- Configure the agent to provide actionable remediation steps rather than just raw error codes.

### 2) Composio Toolset Node
- Provide your PhantomBuster API key within the Composio configuration.
- Ensure the connection scope includes read/write access to agent logs and execution controls.

### 3) Tool Availability
- `list_agents`: Retrieves a comprehensive list of all agents in your account.
- `get_agent_logs`: Fetches detailed execution history for specific agent IDs.
- `restart_agent`: Triggers a new execution cycle for a selected agent.
- `update_agent_settings`: Modifies agent configuration based on diagnostic findings.
