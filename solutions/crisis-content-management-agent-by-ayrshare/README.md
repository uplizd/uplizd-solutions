## Quick Start
### 1) Import the Flow into Uplizd
1. Navigate to the Uplizd marketplace and locate the Crisis Content Management Agent.
2. Click "Import Flow" to add the template to your workspace.
3. Connect your social media accounts within the Composio Toolset configuration.
4. Ensure nodes are correctly linked: **Chat Input → Agent → Composio Toolset → Chat Output**.

### 2) Setup the Nodes
- **Chat Input**: Receives the crisis trigger or manual "pause" command from the user.
- **Agent**: Analyzes the input, determines the scope of the crisis, and formulates the necessary API calls.
- **Composio Toolset**: Executes the specific removal or pause commands on your connected social platforms.
- **Chat Output**: Confirms the successful execution of the crisis management protocol to the user.

### 3) Run the Flow
Use the Playground to test your crisis response:
- `Pause all scheduled posts on Twitter and LinkedIn immediately.`
- `Delete all posts from the last 24 hours containing the keyword "summer-sale".`
- `Check status of all active social media queues and report back.`

## Configuration
### 1) Language Model (Agent Node)
The Agent acts as the decision-making engine for your brand safety.
- Use a high-reasoning model to ensure accurate interpretation of crisis triggers.
- Instruct the agent to prioritize safety over speed when identifying content for deletion.
- Configure the agent to provide a summary report of all actions taken after every execution.

### 2) Composio Toolset Node
- Provide your API keys for the social media platforms (e.g., Twitter, LinkedIn, Facebook).
- Set the connection scope to include "write" and "delete" permissions to allow the agent to manage your content queues.

### 3) Tool Availability
- **Social Media Management APIs**: For pausing and resuming queues.
- **Content Moderation Tools**: For identifying and flagging inappropriate text/images.
- **Notification Services**: For alerting the PR team once the crisis protocol has been initiated.
